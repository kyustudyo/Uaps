🎬 HandleGatherSupplies() 호출 타임라인

Worker가 Crystal을 채취할 때 정확히 무슨 일이 일어나는가?

❓ HandleGatherSupplies()는 언제 호출되나요?

💡 답변 미리보기

Behavior Tree의 GatherSuppliesAction이 자원 채취를 완료했을 때 Event Channel을 통해 자동으로 호출됩니다!

아래에서 5초간의 여정을 따라가봅시다! 🚀

// GatherCommand.cs
public override void Handle(CommandContext context)
{
    Worker worker = context.Commandable as Worker;

    // Crystal을 클릭했으므로
    if (context.Hit.collider.TryGetComponent(out GatherableSupply supply))
    {
        worker.Gather(supply);  // 👈 Worker에게 "Crystal 채취해!" 명령
    }
}

// Worker.cs
public void Gather(GatherableSupply supply)
{
    // Behavior Tree 변수 설정
    graphAgent.SetVariableValue("Supply", supply);
    graphAgent.SetVariableValue("TargetGameObject", supply.gameObject);
    graphAgent.SetVariableValue("Command", UnitCommands.Gather); // 👈 "Gather 모드!"
}

protected override Status OnUpdate()

}
5.00000초
6단계: GatherSuppliesAction - 채취 완료! ✨
2초가 지나서 채취가 완료되고, Event Channel로 이벤트를 발행합니다!
// GatherSuppliesAction.cs (계속)
protected override Status OnUpdate()
{
    // ... (위에서 계속)

    // 👇 2초가 지나서 채취 완료!
    if (gatheringTimer >= gatherTime)
    {
        // 🔥 여기서 Event Channel로 이벤트 발행!
        GatherSuppliesEvent.Value.Event?.Invoke(
            Self.Value,                    // Worker GameObject
            supply.AmountPerGather,        // 10 (채취량)
            supply.Supply                  // Crystal (SupplySO)
        );

        // Blackboard 변수 업데이트
        SupplyAmountHeld.Value += supply.AmountPerGather;  // 0 → 10

        return Status.Success;  // 완료!
    }
}
5.00001초
🔥 7단계: HandleGatherSupplies() 호출! (바로 여기!)
Event Channel이 이벤트를 발행하면, 구독한 함수가 자동으로 호출됩니다!

⚡ 구독 설정 (게임 시작 시 미리 해놨음)

// Worker.cs - Start()에서
protected override void Start()
{
    base.Start();

    // 👇 Event Channel에 구독! (Start에서 1번만)
    if (graphAgent.GetVariable("GatherSuppliesEvent",
        out BlackboardVariable<GatherSuppliesEventChannel> eventChannelVariable))
    {
        eventChannelVariable.Value.Event += HandleGatherSupplies;  // 👈 구독!
    }
}

🎉 호출되는 순간!

// Worker.cs - 바로 이 함수가 호출됨!
private void HandleGatherSupplies(GameObject self, int amount, SupplySO supply)
{
    // 파라미터:
    // self   = Worker GameObject
    // amount = 10 (채취량)
    // supply = Crystal (SupplySO)

    // Event Bus로 다시 발행 → Supplies.cs가 듣고 UI 업데이트!
    Bus<SupplyEvent>.Raise(new SupplyEvent(amount, supply));
}
5.00002초
8단계: Supplies.HandleSupplyEvent() - UI 업데이트 🎉
Event Bus를 통해 자원 관리 클래스가 이벤트를 받고 화면을 업데이트합니다.
// Supplies.cs
private void HandleSupplyEvent(SupplyEvent evt)
{
    // evt.Amount = 10
    // evt.Supply = Crystal

    if (evt.Supply.Equals(mineralsSO))  // Crystal == Minerals?
    {
        Minerals += evt.Amount;  // 100 → 110
        mineralsText.SetText(Minerals.ToString());  // UI 업데이트: "110"
    }
}

🔑 핵심 정리

HandleGatherSupplies()는 언제 호출되나요?

  • Behavior Tree의 GatherSuppliesAction이 자원 채취를 완료했을 때
  • Event Channel을 통해 자동으로 호출됨
  • 구독 설정은 Worker.Start()에서 미리 해놨음 (+= 연산자)

📻 방송국 비유:

  • GatherSuppliesAction: "자원 채취 완료!" 📻 (방송국)
  • Event Channel: 전파 📡 (중계)
  • HandleGatherSupplies: "알겠습니다!" 📻 (라디오 수신)
❓ 그럼 OnUpdate()는 언제 호출되나요?

⏱️ OnUpdate()는 매 프레임마다 호출됩니다!

Unity의 Update()와 비슷하지만, Behavior Tree 노드가 활성화되어 있을 때만 호출됩니다!

🎯 OnUpdate() 호출 시점

📅 3초 ~ 5초 동안 (2초간)

// GatherSuppliesAction.cs
protected override Status OnUpdate()
{
    // 👇 이 함수가 매 프레임마다 호출됨!
    // 60 FPS라면 2초 동안 약 120번 호출!

    gatheringTimer += Time.deltaTime;  // 0.016초씩 증가

    // 3.00초: gatheringTimer = 0.00
    // 3.01초: gatheringTimer = 0.016
    // 3.03초: gatheringTimer = 0.032
    // ...
    // 4.99초: gatheringTimer = 1.99
    // 5.00초: gatheringTimer = 2.00 ✅ 완료!

    if (gatheringTimer >= gatherTime)  // 2초
    {
        // 채취 완료! Event Channel 발행
        GatherSuppliesEvent.Value.Event?.Invoke(...);
        return Status.Success;
    }

    return Status.Running;  // 아직 진행 중
}

🔄 호출 횟수 계산:

  • 60 FPS 기준: 1초에 60번 호출
  • 2초 동안: 약 120번 호출
  • 매 프레임: Time.deltaTime (약 0.016초)만큼 타이머 증가
  • 2초 도달 시: Status.Success 반환 → 노드 완료!

💡 Unity Update()와 차이점

특징 Unity Update() Behavior Tree OnUpdate()
호출 시점 매 프레임 (항상) 노드가 활성화되어 있을 때만
사용처 MonoBehaviour 클래스 Behavior Tree 액션 노드
반환 값 void (없음) Status (Running/Success/Failure)
예시 void Update() Status OnUpdate()

📌 핵심 차이:

  • Update(): 게임이 실행되는 동안 항상 호출
  • OnUpdate(): Behavior Tree 노드가 활성화된 동안만 호출
  • 예를 들어, GatherSuppliesAction은 3초~5초 동안만 활성화되므로
    OnUpdate()도 그 2초 동안만 호출됨!
