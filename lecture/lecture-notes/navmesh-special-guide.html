🏗️ NavMesh의 3가지 핵심 요소

1️⃣ NavMesh Areas (영역)

뜻: "이 지역은 비싸다", "이 지역은 싸다"

예시: 물은 2배 비싸게, 점프는 3배 비싸게

2️⃣ NavMesh Modifier (수정자)

뜻: "이 건물은 영원히 통과 불가"

특징: 게임 시작 전에 미리 설정 (베이킹 시)

3️⃣ NavMesh Obstacle (장애물)

뜻: "이 리소스는 채굴되면 사라져"

특징: 게임 중에 실시간으로 변화

⚔️ Modifier vs Obstacle - 대결!

구분 NavMesh Modifier NavMesh Obstacle
🎯 사용 목적 영구적 장애물
(건물, 벽, 물)		 임시 장애물
(리소스, 파괴 가능한 객체)
⏰ 작동 시점 게임 시작 전
(베이킹 시)		 게임 중
(런타임)
🚀 성능 매우 좋음
(한 번만 계산)		 보통
(실시간 계산)
🔄 변경 가능성 불가능
(리베이킹 필요)		 가능
(즉시 변경)

🎮 실제 게임 상황으로 이해하기

🏠 건물 (Barracks) - Modifier 사용

상황: 플레이어가 건물을 짓는다

문제: 유닛들이 건물을 뚫고 지나가면 안 되죠?

// Modifier를 사용하는 이유 1. 건물은 영원히 그 자리에 있음 2. 성능이 중요 (많은 건물이 있을 수 있음) 3. 변경할 필요가 없음 // 결과 ✅ 유닛들이 건물을 피해서 이동 ✅ 성능 효율적 ✅ 안정적

⛏️ 미네랄 (Minerals) - Obstacle 사용

상황: 유닛이 미네랄을 채굴한다

문제: 채굴 전에는 장애물, 채굴 후에는 통과 가능해야 함

// Obstacle을 사용하는 이유 1. 미네랄은 채굴되면 사라짐 2. 실시간으로 변화해야 함 3. 유연성이 중요 // 채굴 전 미네랄: "나는 여기 있어! 통과 못해! 🚫" 유닛: "알겠어, 돌아서 갈게" // 채굴 후 미네랄: "사라졌어! 이제 통과해도 돼! ✅" 유닛: "좋아, 이제 이 길로 갈 수 있네!"

🔧 실제 코드로 보는 설정

Minerals Prefab 설정

// Minerals에 NavMesh Obstacle 추가 NavMeshObstacle obstacle = minerals.AddComponent<NavMeshObstacle>(); // 설정값들 obstacle.shape = NavMeshObstacleShape.Box; // 박스 모양 obstacle.extents = new Vector3(0.25f, 0.125f, 0.25f); // 크기 obstacle.center = new Vector3(0, -0.5f, 0); // 중심점 obstacle.carve = true; // NavMesh 조각내기 obstacle.carveOnlyStationary = true; // 정지 상태에서만
💡 왜 이런 설정을 하나요?
• carveOnlyStationary = true: 움직이는 동안은 계산하지 않아서 성능 절약
• center = (0, -0.5, 0): 미네랄의 바닥 부분에 맞춤
• extents = (0.25, 0.125, 0.25): 미네랄 크기에 딱 맞게

Gas Prefab 설정

// Gas는 Minerals와 다른 설정 obstacle.extents = new Vector3(0.9f, 0.125f, 0.5f); // 가스는 더 넓고 낮음 obstacle.center = new Vector3(0.15f, 0, 0); // 가스 위치에 맞춤 // 왜 다를까? - Gas는 Minerals보다 넓고 낮음 - 각 리소스의 실제 크기에 맞춰야 함 - 유닛이 정확히 리소스 근처까지 갈 수 있도록

🎯 핵심 포인트 정리

✅ 기억해야 할 3가지

  1. Modifier = 영원한 장애물 (건물, 벽)
  2. Obstacle = 임시 장애물 (리소스, 파괴 가능한 것)
  3. 성능을 고려해서 적절히 선택하기

🤔 "언제 뭘 써야 하지?"

질문: "이 객체가 게임 중에 사라질 수 있나요?"

답변:

  • ✅ Yes → NavMesh Obstacle 사용
  • ❌ No → NavMesh Modifier 사용
